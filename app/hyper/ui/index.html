Getting Started

Evothings consists of three connected parts, Evothings Workbench (the application you are currently running), the Evothings Viewer app running on your mobile device(s) and cloud servers to connect the Workbench and the Viewer applications. You need all three parts to be able to use Evothings.

Run apps on connected Viewers by clicking on the apps listed under Examples and My Apps. The currently running app will be automatically reloaded when you edit and save any of the source code files!

Step 1: Install Evothings Viewer

Install the Evothings Viewer on your mobile device(s), get it from Google Play or Apple App Store.

Step 2: Login to the Workbench using Github/Facebook etc

To run apps on your mobile phones and tablets, login with the same account in both Evothings Workbench and Evothings Viewer. That enables connected Viewers to find the associated Workbench in a secure way.

Go to the Login screen and click the button to login.

Step 3: Login from the Viewer to connect to the Workbench

To connect from Evothings Viewer to the Workbench, you need to login with the same account as you used to login to the Workbench. You also need Internet access on your phone/tablet.

Step 4: How to create a new app

To create your own apps you have three options. Copy one of the Evothings example apps by clicking the button, and modify it for your own purposes, or click the button in the top right to create a basic app from a template.

You can also drag and drop any .html file (typically index.html) to the Workbench window to add an existing app under My Apps.

To add a project with ECMAScript 6 support, drag the project folder or the evothings.json file into the Workbench window. Learn more in the ECMAScript documentation for Evothings.

Arrange apps using drag and drop (the example apps cannot be rearranged though). Click the close icon (x) to delete an app entry. This will NOT delete the application files.

Step 5: Code in your favourite editor end enjoy live reload

Launch the app you wish to work on in connected Viewers using the button. This makes it the active app. When you save any code file for the active app, it will automatically get reloaded on all connected mobile devices.

To edit the code of an app, click the button on the app under My Apps. This will by default open Visual Studio Code to edit the selected application. You can change which editor to use under settings. You can also use the button (click the button first) to open a system file browser for your app. Note that you cannot edit the examples directly, you need to make a copy first.

Edit any of the files and press save to reload the app.

For further details, please visit the online documentation.

Happy coding!

Login

Use Evothings Workbench with the Evothings Viewer mobile app to enjoy a fast development workflow and other features. The Viewer app connects to the Workbench using Evothings proxy servers. Read the Getting Started Guide for detailed first-time instructions. You get the Evothings Viewer mobile app from Google Play or Apple App Store.

To run apps in the Evothings Viewer, you need Internet access and you need to login both here in Evothings Workbench and in the Evothings Viewer.

Login to the Workbench and the Viewer using the same account. This securely connects any number of Viewers to the Workbench. Logging in to another workbench with this account, automatically logs out the previous workbench.

To run the example apps in Evothings Viewer, click the Run buttons on the Examples screen.

Logout from this workbench. This logs you out from this workbench and also logs out connected viewers. If you want to loginto another workbench and keep connected viewers logged in, you can simply login to the other workbench. This will automatically log you out from this workbench.

Evothings has the ability to build your application for Android, and iOS is coming soon!

To build your application you simply press the button on your app under My Apps and then hit the button that appears, and follow the instructions.

The build mechanism uses a virtual preconfigured Cordova build process which requires only minimal input from you to work automatically. It will generate an Android APK file that can be installed on Android devices or published to Google play.

The Workbench will guide you to download two external tools needed - VirtualBox and Vagrant - both very easy to install. Then it will proceed with automatically setting up a virtual build environment and finally start the build process. These steps are only performed once, any subsequent builds are just a single click.

The build process will run in the background and a live log of the process is shown here below. When the process is finished, you will be alerted and you will find a link to the Android APK file at the top of the Build tab. Click and save it, this APK can now be sent to, and installed on your Android devices or be published at Google play.
